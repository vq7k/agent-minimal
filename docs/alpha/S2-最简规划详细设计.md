---

## 1. 目标

S2 只解决一件事：当路由结果为 `plan`，且用户已经给出明确出游意图时，alpha 能返回一份可读的一日游行程。

本片完成后：

- `out_of_scope`：保持 S1 固定拒答。
- `qa`：保持 S1 普通聊天兜底。
- `plan`：生成一份包含七段信息的一日游方案。

七段为：

1. 概览
2. 时间轴
3. 餐饮
4. 路线
5. 预算
6. 贴士
7. 声明

---

## 2. 非目标

S2 不做以下内容：

- 不做缺城市/缺日期追问。
- 不拆 `planner` / `capabilities`。
- 不接真实天气、景点、餐饮 API。
- 不新增 `status` 事件。
- 不改前端。
- 不做多轮偏好记忆或增量调整。

理由：S2 的用户可见增量是“给城市+日期能得到行程”。缺失追问属于 S3；过程可见属于 S5。

## 4. 行程模块设计

### 4.1 对外接口

```python
def generate(messages: list[dict]) -> str:
    ...
```

返回值是最终可直接流式输出的文本。

### 4.2 内部流程

```text
generate(messages)
  ├─ data = llm.generate_structured(prompt, schema_hint, validate)
  ├─ if data is None:
  │    └─ return fallback_text(messages)
  └─ return render(data)
```

### 4.3 schema

LLM 结构化结果必须是 dict，且包含七个字段：

```python
REQUIRED_SECTIONS = (
    "overview",
    "timeline",
    "dining",
    "route",
    "budget",
    "tips",
    "disclaimer",
)
```

字段要求：

- 每个字段必须存在。
- 每个字段必须是非空字符串。
- 不要求 timeline 是数组；S2 为降低实现复杂度，先用可读文本承载。

---

## 5. Prompt 设计

Prompt 目标是让 LLM 一次产出完整行程，不拆工具：

```text
你是 alpha，一日游规划助手。
请根据完整 messages，为用户生成一份一日游方案。

要求：
- 只规划一天。
- 内容要具体、可读、适合直接展示给用户。
- 必须包含概览、时间轴、餐饮、路线、预算、贴士、声明。
- 声明必须表达“信息仅供参考，出行前请确认实时信息”。
- 当前不接真实天气/票务/餐厅 API，不要假装实时查询。

只输出 JSON 对象，字段为：
overview/timeline/dining/route/budget/tips/disclaimer
```

实现时把完整 `messages` 作为 JSON 文本附在 prompt 后面，使用 `ensure_ascii=False`。

---

## 6. 文本渲染

`render(data)` 输出 Markdown 风格文本：

```text
## 一日游方案

### 概览
...

### 时间轴
...

### 餐饮
...

### 路线
...

### 预算
...

### 贴士
...

### 声明
...
```

S2 不要求前端特殊渲染；当前聊天气泡按普通文本展示即可。

---

## 7. 降级策略

当 `generate_structured()` 返回 `None`：

- 不抛异常。
- 返回一份朴素文本兜底。
- 兜底文本必须包含“信息仅供参考”。

兜底文案示意：

```text
我可以先给你一个简版一日游安排：上午选择城市核心景点，中午在附近用餐，下午安排第二个景点或城市漫步，傍晚返程。信息仅供参考，出行前请确认天气、营业时间和交通情况。
```

这样即使结构化规划失败，`plan` 分支仍有用户可见结果。

---

## 8. `chat()` 编排设计

S2 后的 `chat()`：

```text
chat(messages)
  ├─ kind = intent.route(messages)
  ├─ out_of_scope → fixed text + done
  ├─ plan → itinerary.generate(messages) + done
  └─ qa → llm.stream_text(messages) + done
```

`plan` 分支本片不做 token 级流式。直接生成完整文本后发一个 `text` 事件即可；S5 再做过程可见和分步输出。

## 9. 代码实现草案

本节给出可直接落地的代码形态。实现时允许做小幅命名调整，但不要扩大 S2 范围。

### 9.1 `agents/alpha/_core/itinerary.py`

```python
"""S2 最简行程生成：一次 LLM 产出整份一日游方案。"""

import json

from agents.alpha._core import llm

REQUIRED_SECTIONS = (
    "overview",
    "timeline",
    "dining",
    "route",
    "budget",
    "tips",
    "disclaimer",
)

_SCHEMA_HINT = """{
  "overview": "概览",
  "timeline": "一天时间轴",
  "dining": "餐饮建议",
  "route": "路线安排",
  "budget": "预算估算",
  "tips": "出行贴士",
  "disclaimer": "信息仅供参考声明"
}"""

_HEADINGS = {
    "overview": "概览",
    "timeline": "时间轴",
    "dining": "餐饮",
    "route": "路线",
    "budget": "预算",
    "tips": "贴士",
    "disclaimer": "声明",
}


def _validate_itinerary(data: dict) -> bool:
    return all(isinstance(data.get(key), str) and data[key].strip() for key in REQUIRED_SECTIONS)


def _prompt(messages: list[dict]) -> str:
    return f"""你是 alpha，一日游规划助手。
请根据完整 messages，为用户生成一份一日游方案。

要求：
- 只规划一天。
- 内容要具体、可读、适合直接展示给用户。
- 必须包含概览、时间轴、餐饮、路线、预算、贴士、声明。
- 声明必须表达“信息仅供参考，出行前请确认实时信息”。
- 当前不接真实天气/票务/餐厅 API，不要假装实时查询。

messages:
{json.dumps(messages, ensure_ascii=False)}
"""


def _fallback_text() -> str:
    return (
        "我可以先给你一个简版一日游安排：上午选择城市核心景点，"
        "中午在附近用餐，下午安排第二个景点或城市漫步，傍晚返程。"
        "信息仅供参考，出行前请确认天气、营业时间和交通情况。"
    )


def render(data: dict) -> str:
    parts = ["## 一日游方案"]
    for key in REQUIRED_SECTIONS:
        parts.append(f"### {_HEADINGS[key]}\n{data[key].strip()}")
    return "\n\n".join(parts)


def generate(messages: list[dict]) -> str:
    data = llm.generate_structured(_prompt(messages), _SCHEMA_HINT, _validate_itinerary)
    if data is None:
        return _fallback_text()
    return render(data)
```

### 9.2 `agents/alpha/__init__.py` 接入点

只新增 `itinerary` import 和 `plan` 分支：

```python
from agents.alpha._core import intent, itinerary, llm


def chat(messages: list[dict]) -> Iterator[str]:
    kind = intent.route(messages)
    if kind == "out_of_scope":
        yield _text(OUT_OF_SCOPE_REPLY)
        yield _done()
        return

    if kind == "plan":
        yield _text(itinerary.generate(messages))
        yield _done()
        return

    for delta in llm.stream_text(messages, system=SYSTEM_PROMPT):
        yield _text(delta)
    yield _done()
```

## 13. 完成标准

S2 完成时应满足：

- `plan` 分支不再走普通聊天兜底，而是进入行程生成。
- 行程结构化结果七段齐才进入渲染。
- 结构化失败有朴素文本降级。
- S1 的 `qa/out_of_scope` 行为不回退。
- 不改前端，不引入新依赖。
