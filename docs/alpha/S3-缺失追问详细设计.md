---

## 1. 目标

S3 只解决一件事：

`plan` 分支在进入 S2 行程生成前，先判断是否缺少一日游规划的必要信息；
缺少则一次性追问，信息齐全才进入 `itinerary.generate()`。

本片完成后：

- `out_of_scope`：保持 S1 固定拒答。
- `qa`：保持普通聊天兜底。
- `plan` 且缺城市/日期：返回追问文案。
- `plan` 且城市/日期齐全：进入 S2 最简规划。

---

## 2. 非目标

S3 不做以下内容：

- 不做复杂多轮状态机。
- 不持久化用户信息。
- 不抽取偏好记忆。
- 不改前端。
- 不新增 `status` 事件。
- 不验证真实日期是否合法。

理由：S3 的用户可见增量是“信息不全时问清楚”。偏好记忆属于 S4；过程可见属于 S5。

## 4. 模块设计

### 4.1 数据形状

```python
Slots = dict[str, str | None]

REQUIRED_FIELDS = ("city", "date")
```

S3 只把 `city/date` 当作必要字段。
其他字段如 `companions/budget/interest/pace` 先不参与缺失判断，避免扩大范围。

### 4.2 对外接口

```python
def extract(messages: list[dict]) -> dict:
    ...


def missing(slots: dict) -> list[str]:
    ...


def ask(missing_fields: list[str]) -> str:
    ...
```

### 4.3 内部流程

```text
extract(messages)
  ├─ data = llm.generate_structured(prompt, schema_hint, validate)
  ├─ if data is None:
  │    └─ return {"city": None, "date": None}
  └─ return data

missing(slots)
  └─ 返回 city/date 中为空的字段

ask(missing_fields)
  └─ 生成一次性追问文案
```

## 5. Prompt 设计

```text
你是 alpha 的出游需求理解器。
请从完整 messages 中抽取一日游规划必要信息。

只需要抽取：
- city：目的地城市或地点
- date：出游日期、星期、相对日期或时间表达

如果没有提到，字段填 null。
只输出 JSON 对象：{"city": string|null, "date": string|null}
```

实现时把完整 `messages` 作为 JSON 文本附在 prompt 后面，使用 `ensure_ascii=False`。

---

## 6. 校验与降级

### 6.1 校验

结构化结果必须满足：

- 是 dict。
- 包含 `city` 和 `date`。
- 每个字段值只能是 `str` 或 `None`。

### 6.2 降级

`generate_structured()` 返回 `None` 时：

```python
{"city": None, "date": None}
```

理由：理解失败时不应直接编造城市/日期；追问用户比生成错误行程更稳。

---

## 7. 追问文案

追问要短，一次性问清缺失字段。

| 缺失字段 | 文案 |
|---|---|
| `city` | `你想去哪个城市或地点玩？` |
| `date` | `你计划哪天出发？` |
| `city,date` | `我需要先确认目的地和日期：你想去哪个城市或地点，计划哪天出发？` |

S3 不要求 LLM 生成追问文案。固定文案更稳定，也更容易测试。

## 9. 代码实现草案

### 9.1 `agents/alpha/_core/understanding.py`

```python
"""S3 需求理解：抽取 city/date 并判断缺失。"""

import json

from agents.alpha._core import llm

REQUIRED_FIELDS = ("city", "date")
_SCHEMA_HINT = '{"city": "string|null", "date": "string|null"}'


def _validate_slots(data: dict) -> bool:
    return all(key in data and (data[key] is None or isinstance(data[key], str)) for key in REQUIRED_FIELDS)


def _prompt(messages: list[dict]) -> str:
    return f"""你是 alpha 的出游需求理解器。
请从完整 messages 中抽取一日游规划必要信息。

只需要抽取：
- city：目的地城市或地点
- date：出游日期、星期、相对日期或时间表达

如果没有提到，字段填 null。

messages:
{json.dumps(messages, ensure_ascii=False)}
"""


def extract(messages: list[dict]) -> dict:
    data = llm.generate_structured(_prompt(messages), _SCHEMA_HINT, _validate_slots)
    if data is None:
        return {"city": None, "date": None}
    return data


def missing(slots: dict) -> list[str]:
    return [key for key in REQUIRED_FIELDS if not slots.get(key)]


def ask(missing_fields: list[str]) -> str:
    fields = set(missing_fields)
    if fields == {"city"}:
        return "你想去哪个城市或地点玩？"
    if fields == {"date"}:
        return "你计划哪天出发？"
    return "我需要先确认目的地和日期：你想去哪个城市或地点，计划哪天出发？"
```

### 9.2 `agents/alpha/__init__.py` 接入点

```python
from agents.alpha._core import intent, itinerary, llm, understanding

        return
    yield _text(itinerary.generate(messages))

```

## 11. 单元测试用例

对应文件：`tests/test_alpha_s3.py`。

| ID | 用例 | 关键断言 | RED 预期 |
|---|---|---|---|
| S3-1 | 城市和日期齐全 | `missing(slots) == []` | 缺 `understanding` |
| S3-2 | 缺城市 | 追问包含“城市或地点” | 缺 `understanding` |
| S3-3 | 缺日期 | 追问包含“哪天” | 缺 `understanding` |
| S3-4 | 城市和日期都缺 | 一次性追问目的地和日期 | 缺 `understanding` |
| S3-5 | `plan` 缺信息 | 输出追问；不调用 `itinerary.generate` | 直接进 itinerary |
| S3-6 | `plan` 信息齐全 | 先判槽位，再输出行程文本 | 未接 understanding |

Mock 约定：

- S3-1 mock `llm.generate_structured -> {"city":"杭州","date":"周六"}`。
- S3-2/S3-3/S3-4 直接传入不同缺失形态的 slots。
- S3-5 mock `intent.route -> plan`，`itinerary.generate` 设为报错函数。
- S3-6 mock `intent.route -> plan`，`itinerary.generate -> "行程文本"`。

测试不覆盖：

- 日期合法性。
- 多轮状态机。
- 偏好抽取。
- 真实 LLM 理解准确率。

## 13. 完成标准

S3 完成时应满足：

- `plan` 分支先判断 city/date 缺失。
- 缺 city/date 时返回固定追问，不调用 `itinerary.generate()`。
- city/date 齐全时保留 S2 行程生成。
- S1 的 `qa/out_of_scope` 行为不回退。
- 不改前端，不引入新依赖。
