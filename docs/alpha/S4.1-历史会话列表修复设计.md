---

## 1. 问题

S4 已实现“刷新后当前会话历史仍在”，但页面暴露了 `New` 按钮。

当前行为：

```text
点击 New
  └─ 生成新 conversation_id
  └─ localStorage 覆盖为新 conversation_id
  └─ 旧 conversation_id 仍在 DB
  └─ 前端没有入口找回旧 conversation_id
```

这会造成误导：用户以为是“新建会话”，但旧会话在页面上不可见，等同于被隐藏。

结论：这是 S4 当前交互的 bug，不是 S5 记忆功能。

---

## 2. 目标

修复后：

- `New` 不会让旧会话失去入口。
- 页面能看到最近历史会话。
- 点击历史会话能切回并恢复该会话消息。
- 刷新后仍打开当前选中的会话。

---

## 3. 非目标

- 不做会话重命名。
- 不做删除会话。
- 不做搜索。
- 不做分页。
- 不做用户体系；所有会话仍是当前环境下的 alpha 会话。

理由：当前 bug 是“旧会话不可见”，最小修复是列表和切换。

---

## 4. 后端设计

### 4.1 storage 接口

新增：

```python
def list_conversations(limit: int = 20) -> list[dict]:
    ...
```

返回形状：

```json
[
  {
    "conversation_id": "uuid",
    "title": "周六去杭州",
    "updated_at": "2026-06-06T10:00:00+08:00"
  }
]
```

字段来源：

| 字段 | 来源 |
|---|---|
| `conversation_id` | `alpha_messages.conversation_id` |
| `title` | 该会话第一条 `user` 消息，截断到 24 个中文字符以内；没有 user 则显示 `未命名会话` |
| `updated_at` | 该会话最新消息的 `created_at` |

排序：

```text
最新消息时间倒序，最多 20 条
```

### 4.2 API

新增：

```text
GET /agents/alpha/conversations
```

响应：

```json
{
  "conversations": [
    {
      "conversation_id": "uuid",
      "title": "周六去杭州",
      "updated_at": "2026-06-06T10:00:00+08:00"
    }
  ]
}
```

## 5. 前端设计

在 Alpha 页头增加轻量会话列表，位置和行为：

```text
Alpha 标题区
  ├─ 当前会话标题/状态
  ├─ New 按钮
  └─ 最近会话列表（最多 20 条）
```

行为：

| 操作 | 结果 |
|---|---|
| 页面加载 | 读取当前 `conversationId`，同时拉取最近会话列表 |
| 点击历史会话 | 保存该 `conversationId` 到 localStorage，并加载该会话 messages |
| 点击 New | 新建 `conversationId`，展示初始消息；旧会话仍保留在列表中 |
| 发送消息成功 | 刷新最近会话列表，让当前会话出现在顶部 |

UI 原则：

- 不把列表做成复杂侧边栏；当前页面很小，先用标题区下方的紧凑列表。
- 历史项显示一行摘要，避免挤压聊天区域。
- 当前会话高亮。

## 6. 单元测试用例

后端：

| 用例 | 断言 |
|---|---|
| `storage.list_conversations()` | 按最近消息倒序返回 `conversation_id/title/updated_at` |
| `GET /agents/alpha/conversations` | 返回 `{"conversations": [...]}` |

前端 API：

| 用例 | 断言 |
|---|---|
| `fetchAlphaConversations()` | GET `/agents/alpha/conversations` |
| `fetchAlphaConversations()` | 响应必须包含数组 `conversations` |

页面人工验收：

| 场景 | 期望 |
|---|---|
| 会话 A 聊两轮，点 New，新建 B | A 仍出现在历史会话列表 |
| 点击 A | 消息恢复为 A |
| 刷新页面 | 当前选中的会话仍保持 |
