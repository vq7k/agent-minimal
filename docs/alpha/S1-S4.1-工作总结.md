---

## 1. 本轮结论

本轮把 alpha 从“能聊天的 agent”推进到“有清晰能力边界、能做最简一日游规划、能缺失追问、能用 DB 持久化多会话、刷新后恢复、New 后旧会话可见并可切回”的版本。

当前不继续做 S5。S5 的“偏好微调 / 记忆”需要在已有 DB 会话历史基础上重新设计，不能把“单次上下文里已有历史消息”包装成长期记忆。

## 4. 本轮重要决策

| 决策 | 结论 |
|---|---|
| 自动化范围 | 克制，只测路由、schema、SSE、DB、已踩坑边界 |
| ruff | 去掉，不作为日常开发验证项 |
| UI 测试 | S4.1 页面列表走人工验收，不为小功能引入 `jsdom/testing-library` |
| mock 边界 | 只 mock LLM / 网络 IO，不 mock 业务流程本身 |
| DB 方案 | 使用共享 PostgreSQL，S4 只做消息表，不提前做用户画像 |
| 记忆定义 | 单次会话历史不是长期记忆；S5 需要单独设计 |
